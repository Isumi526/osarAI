> toC営業マン向け 顧客管理 × AI戦略相談 アプリ
> 作成：2026-06 ／ ステータス：MVP叩き台

## 1. 背景・目的

株式会社LiberalLIFE（以下 LL）は金融・保険／不動産／通信／生活インフラ／転職など15事業を展開し、約250名の営業代理店を抱える。代理店向け福利厚生として提携先サービスを特別価格で提供する仕組みがあり、本プロダクトはその提携サービスの一つとして月額提供する。

LLは代理店に対して統一の顧客管理基盤を持たず、各自任せ。結果として**大多数が顧客管理をしていない**状態。本プロダクトは、toC営業マンの顧客管理と営業戦略をAIで支援し、最終的には人脈マッチングまで内包する「営業のワンストップ基盤」を目指す。

**重要な前提：osarAIは運営者（亥角）所有の独立プロダクトであり、LLは販路の1つに過ぎない。** データ・顧客ネットワーク・知的財産はすべて運営者側に帰属する。LLは最初の大口導入先（アンカーテナント）と位置づけ、汎用マルチテナントSaaSとして設計し、他のtoC営業組織へ横展開する。→ LL専用カスタムは避け、組織単位でデータ分離する構造を初手から取る。データ所有権を運営者に置くことで、Phase2のマッチング資産・横展開の自由度・対LL交渉力をすべて確保する。

## 3. コンセプト / 提供価値

**タグライン：「忙しくても、人を大切にできる自分に。」**
中心UX：人と会ったあと5分の、AI対話"おさらい"。

**中心思想：**
営業成果とは、結局「どれだけ人に喜ばれ、気に入られ、役に立てたか」の積み重ね。数字はその"結果"であって、"目的"ではない。本当に大事なのは、目の前の商談を取り切ることより──

- お客さんと話したことを覚えている
- それに合わせて、後日ちゃんと連絡する
- 望まれている人を、繋いであげる

という「人を大切にし続けること」。誰もがそうありたいが、目の前の忙しさで手が回らない。**それを、5分の習慣 × AI が肩代わりする。** osarAIは"ゴリゴリ営業ツール"ではなく、**誠実に人と向き合いたい人のための「記憶と気配り」の外部装置**。ターゲットは短期で刈り取る人ではなく、長く信頼で生きていく人。

**訴求の方針：**
- 言葉づかいは狩猟的（逃さない・取り切る・追客）を避け、関係的（覚えている・気にかける・繋ぐ・大切にする）に寄せる。
- 成果・実利は否定しないが、"目的"ではなく"人を大切にした結果"として位置づける。
- 「営業」という職業で括らず、行為・対象ベースで広く受ける（営業/対面サービス業すべて）。
- 継続が生命線：5分・AIが聞いてくる・会ったあとのプッシュ、で習慣化＝アクティブ率＝MRR。
- Phase2マッチングも「案件をぶんどる」ではなく「**望まれている人を繋いであげる＝人助け**」の文脈で語る。

**対象の射程（将来）：**
osarAIのコアは「営業」ではなく「**人を相手にするリピート商売の、顧客記憶＆関係維持**」。営業はその一例で、パーソナルトレーナー／美容師／セラピスト／コーチ／士業など、継続的に人と会い「相手を覚えておくこと」が成果に直結する業種すべてが射程。サービス名・コンセプトを業種非依存にしてあるため横展開しやすい。
→ ただし **MVPはtoC営業（LL）に集中**。業種展開はマッチング同様、横展開フェーズで解放する。設計上は顧客カードを業種非依存の汎用スキーマにしておき、業種別はカスタム項目で着せ替え可能にする。

## 4. スコープ

### MVPに含む（Phase 1）
- **AI対話ガイド入力**（メイン）＋ 録音・録画の取り込み（サブ）→ AI解析・整理
- 顧客管理（カード・タイムライン）
- AI戦略相談（全体 or 顧客指定で次アクション提案）
- リーダー：配下メンバーのデータ集約閲覧
- 認証・組織/プラン管理（マルチテナント基盤）

### MVP対象外（Phase 2 以降）
- **人脈マッチング**（データが溜まってから点火する"二段ロケット"。組織内マッチングから）
- 組織横断マッチング（同意・フィー設計クリア後のオプション）
- 紹介フィーの可視化・自動計算
- LLの15事業クロスセル連携
- フル音声対話（ハンズフリー）、録音取得の自動化（ネイティブ録音／Zoom bot）

> MVPの肝は「**AIが聞いてくる→答えるだけで勝手に整理される**」体験。フォーム入力をさせない。摩擦ゼロでアクティブ率＝継続率＝MRRを担保することが最優先。機能過多は厳禁。

## 5. 機能要件

### F-01 顧客管理（CRUD）
- 顧客の登録・編集・一覧・詳細表示
- AI解析で抽出された情報（ニーズ／温度感／次アクション等）を顧客に紐付け
- AC:
  - [ ] 顧客を新規作成・編集・削除できる
  - [ ] 顧客詳細に「対応履歴（タイムライン）」が表示される
  - [ ] 一覧をステータス／温度感で絞り込みできる

### F-02 AI対話ガイド入力（★メイン入力）
toC営業は雑談・お茶アポ中心で、あからさまな録音録画はしにくい。よってメインの入力経路は**録音ではなく「AIが聞いてくる対話」**とする。ユーザーはフォームを埋めるのではなく、思い出して喋る／雑に書くだけ。構造化は裏でAIが行う。

**仕組み**
- AIインタビュアーがアポ後にヒアリング（例：「◯◯さんのアポどうやった？」→足りない項目を追い質問）
- ユーザーの発話／入力からAIが顧客カードの項目（ニーズ／温度感／家族構成／課題／次アクション等）を自動抽出・構造化
- 最後にサマリを提示し、ユーザーは確認・微修正だけ

**効かせる仕掛け**
- 過去データを使った追い質問（2回目以降は差分・進展を聞く＝使うほど賢く＝ロックイン）
- アポ後プッシュで「能動入力→受動回答」化（記憶が新鮮なうちに通知→タップして答えるだけ）
- 自由入力は音声、定型（温度感など）はタップのハイブリッド
- 雑に喋ってもAIが要約・整形するため入力プレッシャーが低い

**入力形式（MVP）**：チャット形式 ＋ 音声入力（A）を両方サポート
- A=音声入力：チャットUIのまま、入力だけ音声（ブラウザ依存回避のためサーバー側STT＝音声をGeminiでテキスト化を推奨）
- B=フル音声対話（ハンズフリー、AIも音声で返す。移動・運転中向け）は **Phase 2**

- AC:
  - [ ] AIが対話形式で顧客情報をヒアリングできる
  - [ ] 発話／入力から項目が自動抽出され、顧客カードに反映される
  - [ ] 2回目以降は既存データを踏まえた質問になる
  - [ ] テキスト入力・音声入力の両方で回答できる
  - [ ] 温度感など定型項目はタップで素早く入力できる
  - [ ] 生成されたサマリをユーザーが確認・修正できる
  - [ ] （任意）アポ終了後にヒアリングを促す通知を出せる

### F-03 録音・録画の取り込み（サブ／録れる時だけ）
録音録画ができた場合のオプション入力。入口は違っても、その先はF-02と同じ「文字起こし → AI解析 → 顧客に紐付け」パイプラインに合流する（**入口は複数・処理は共通**）。
- 対面：スマホで録った音声を取り込み
- オンライン：Zoom録画ファイルを取り込み
- ※自動化（アプリ内ネイティブ長時間録音／Zoom bot連携）は Phase 2
- AC:
  - [ ] 音声ファイル（対面録音／Zoom録画）を取り込める
  - [ ] 文字起こし＋要約が生成され、対象顧客に保存される
  - [ ] 抽出結果をユーザーが修正できる
  - [ ] 解析中の進捗が分かる（非同期処理）
  - [ ] 入力経路の種別を記録する（将来の経路追加に備える）

> 設計上の含意：メイン入力が対話・録音がサブのため、MVPで録音デバイス化は不要。**配信形態はWeb/PWAで成立**（Capacitorは対面ネイティブ録音／フル音声対話を本格化するPhase 2で導入）。

### F-04 AI戦略相談チャット
- AIチャットに次アクションを相談。蓄積された顧客データを根拠に提案
- スコープ選択：**全体**（ポートフォリオ視点）／**顧客指定**（個別深掘り）
- AC:
  - [ ] チャットで自由に相談できる
  - [ ] 「全体」モードで全顧客を俯瞰した提案が返る
  - [ ] 「顧客指定」モードで特定顧客に絞った提案が返る
  - [ ] 提案が実データ（顧客の状況）を踏まえている

> 留意：初期はデータが薄くコールドスタートになる。汎用の営業ナレッジ＋薄い顧客データで成立させ、蓄積後に個別最適へ移行する設計。

### F-05 リーダー集約ビュー
- リーダーが配下メンバーの顧客状況・活動を横断で閲覧
- AC:
  - [ ] 配下メンバー一覧と各自の主要指標（顧客数・活動量等）を見られる
  - [ ] メンバー個別の顧客リスト／履歴をドリルダウンできる
  - [ ] リーダーは編集ではなく閲覧主体（権限分離）

### F-06 認証・組織・プラン・課金
- 組織（テナント）単位のデータ分離、ユーザーのロール（member / leader）管理
- 課金は**個人課金（Stripeサブスク）**。**登録・課金はWebで完結し、アプリは利用専用**（アプリ内課金導線を一切持たない＝IAP回避）。
- **3層プラン（Good-Better-Best）**：Light ¥1,980（おさらい＋顧客管理／AI相談 月10回）／**Standard ¥3,980（本命：AI相談無制限＋録音取り込み）**／Pro ¥6,980（＋リーダー集約＋Phase2マッチング優先）。Standardへ誘導（デコイ＝Light、アンカー＝Pro）。
- 課金モデルは**14日カード先取りトライアル**：登録時にApple Pay／Google Pay／カードで決済情報を登録 → 14日無料 → 自動課金。Apple Pay/Google PayはStripe Checkoutに載せてワンタップ体験＋低手数料（3%台）を両取り。
- **チャネル割引（B2B2C）**：定価は据え置き、チャネル別の**額引き**プロモコードで割引（定価アンカー維持＋流入トラッキング、割引後も末尾980で統一）。**LL福利厚生価格＝Light¥1,480／Standard¥2,980／Pro¥4,980（各¥500/¥1,000/¥2,000引き＝約25〜29%OFF）**、一般提携は割引額を浅く設定。負担はまずosarAI飲み（原価が安く粗利9割超のため武器化可能）、大口は将来チャネル補填を交渉。
- **プライスラダー**：Phase2マッチングで定価を1段上げる際、既存ユーザー（割引組含む）は旧価格に永久ロック＝値上げ反発回避＋「今入る理由」。
- 導線：LL福利厚生で案内（割引コード入り登録リンク）→ Webで登録＆トライアル開始 → アプリDL → ログイン → 14日後に自動課金
- AC:
  - [ ] サインアップ／ログインできる
  - [ ] ユーザーは必ず1組織に所属する
  - [ ] ロールにより閲覧範囲が変わる（member＝自分、leader＝配下含む）
  - [ ] 他組織のデータには一切アクセスできない（テナント分離）
  - [ ] 登録・課金はWeb（Stripe Checkout）で行う
  - [ ] モバイルアプリ内に課金・価格・外部決済への導線を表示しない（ストア審査対策）
  - [ ] 未契約／解約ユーザーはアプリ機能が制限される

## 6. 非機能要件

- **スマホファースト**：ペルソナ1（おさらいアプリ）はモバイルアプリ運用。リーダーはWeb管理画面をPC併用想定。
- **配信形態**：
  - ユーザー向け「おさらいアプリ」＝ **Capacitor（モバイルアプリ／iOS・Android）**、中身は React。
  - リーダー向け管理画面・LP＝ **Next.js（Web）**。
  - 純Web/PWAではなくモバイルアプリを採る理由は **プッシュ通知**。本プロダクトは「アポ後の"おさらい"習慣化＝アクティブ率＝MRR」が生命線で、その引き金がアポ後プッシュ。PWAのプッシュ（特にiOS）は弱く、習慣化の肝を落とせない。Capacitorならネイティブプッシュ（FCM/APNs）を押さえつつWeb（React）資産を活かせる。
  - ストア配布の障壁は、LL福利厚生＝組織が一括インストール案内できるため許容範囲。
- **プッシュ通知（習慣化の中核）**：アポ終了後に「おさらい」を促すプッシュを送り、能動入力→受動回答に変える。到達率を確保できるネイティブプッシュ前提。
- **入力経路の抽象化**：AI対話／対面録音／Zoom録画／手入力など複数の入口を、共通の「音声・テキスト → 文字起こし → AI解析」パイプラインに合流させる。経路追加で処理側を作り直さない構造にする。
- **文字起こし／AI解析**：長尺音声（30分〜）に備えチャンク分割＋非同期処理。原価は試算済み（標準利用で1ユーザー月¥20〜100規模）。録音乱用に備え1録音あたりの上限のみ設定。
- **マルチテナント**：組織単位の論理分離（Supabase RLS）。横展開の前提条件。
- **コンプラ（要詰め／下記オープン論点へ）**：
  - 商談録音は相手の会話を含む。対面・オンラインとも録音同意の扱いが重い点に留意。
  - 金融・保険商材の会話録音は、顧客の録音同意・個人情報保護法・業法側の記録規制が絡む可能性。MVPでも最低限の利用規約／注意喚起を用意。
- **セキュリティ**：認証はSupabase Auth、録音・個人情報は適切に保護。URLに個人情報を載せない。

## 7. 想定技術スタック

| 層 | 採用（MVP） |
|----|---------|
| モバイルアプリ（おさらい・ユーザー向け） | **React（Vite）+ Capacitor**（iOS / Android） |
| Web（リーダー管理画面・LP） | **Next.js（App Router / SSR）** |
| プッシュ通知 | FCM / APNs（Capacitor 経由） |
| 課金 | **Stripe（サブスク／Web完結、Apple Pay・Google Pay対応）**。3層プラン（Light¥1,980／Standard¥3,980／Pro¥6,980）＋14日カード先取りトライアル→自動課金。チャネル割引は額引きプロモコード（LL価格＝Light¥1,480／Standard¥2,980／Pro¥4,980）。アプリ内課金（IAP）は行わない＝Apple/Google税を回避 |
| バック / DB | Supabase（Postgres + Auth + Storage + RLS） |
| 音声取り込み | ファイル取り込み（対面録音／Zoom録画） |
| 音声入力（STT） | サーバー側STT（音声→Geminiでテキスト化／ブラウザ依存を回避） |
| 文字起こし / AI解析 | Gemini（チャンク分割・非同期） |
| AI対話・チャット | Claude / Gemini いずれか（対話・相談の質で選定） |
| ホスティング | Vercel（Next.js Web） |

**Phase 2で追加**：フル音声対話（ハンズフリー、リアルタイム音声API＝Gemini Live／Realtime系）、対面ネイティブ長時間録音（Capacitorプラグイン）、Zoom連携（録画API取り込み or ミーティングbot）。

> 設計方針：モバイル＝Capacitor+Reactでネイティブプッシュ＆将来の録音/音声を確保、Web管理画面・LP＝Next.jsでSSRの強みを活かす。Supabaseを共通バックエンドに、React実績（モバイル）とNext.js実績（Web）を両取りできる構成。

```
  └ users            … role: member | leader, org_id

              └ ai_messages
```

> マッチング（Phase2）用に、`customers` のニーズを抽象タグ化する `customer_needs_tags` を将来追加できる余地を残す。

## 10. 着手前に決めるべきオープン論点

1. **配信形態** → **Capacitorモバイルアプリ（React）＋Next.js Web管理画面で決着**（プッシュ通知＝習慣化が肝のため）。
2. **課金主体** → **個人課金（Stripeサブスク）で決着**。登録・課金はWeb完結、アプリは利用専用（IAP回避）。
3. **契約・データ所有権** → **運営者（亥角）所有で決着**。LLは販路の1つ。データ・ネットワーク・IPはすべて運営者帰属。
